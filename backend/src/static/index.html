Admin Users

Browser sessions are now subject to the same ACL system as HMAC clients. Passwords still live in admin.htpasswd (edit via create_admin.py); ACLs live in Redis and are edited here.

Users without a provisioned ACL row see no KV. Run python migrate_admin_acls.py on the server to seed defaults, or grant here row-by-row.

Catalogs

Published app catalogs — the human-readable descriptor of each app's tunable variables. Hand-maintained by app owners; device firmware is the arbiter.

Published apps 0

One row per app with a catalog stashed at _catalog/<app>. Published via stra2us catalog publish.

Devices are HMAC clients with access to this app's namespace — ACL prefix <app>, wildcard *, or a deeper <app>/… sub-prefix. Per-device override values join on the client ID.

Every key written under this app's prefix, catalog-tracked or not. Off-catalog keys (drift) are flagged — the firmware can still read them, but they bypass the catalog contract. Read-only; edits go through the Variables tab so invariants stay intact.

Backup & Restore

Export or import all client credentials. Protect your backup with the same care as a password manager file.

⚠ Security Warning: Backup files contain raw HMAC secrets in plaintext. Anyone with this file can authenticate as any of your devices. Store it securely and never commit it to version control.

Download Backup

Downloads a stra2us_backup.json file containing all client IDs, secrets, and ACLs.

Restore from Backup

Upload a previously downloaded backup file. Existing clients will be skipped unless you enable Force Overwrite.
